KNOWN LIMITATIONS (INTENTIONAL - PHASE 1 SCOPE)
================================================================================

1. Frame Buffer Persistence
   - In-memory only (reset on restart)
   - Mitigation: Add Redis in Phase 2
   - Impact: Low (frames updated every 100ms)

2. Audio Polling
   - 150ms polling interval
   - Mitigation: WebSocket in Phase 2
   - Impact: Low (<300ms latency acceptable)

3. Simple Token Auth
   - Bearer token only
   - Mitigation: JWT with expiry in Phase 2
   - Impact: Acceptable for Phase 1

4. Single Server
   - No built-in clustering
   - Mitigation: Load balancer + nginx
   - Impact: Horizontal scaling still works

These are not bugs - they're intentional Phase 1 design decisions.

SUPPORT & TROUBLESHOOTING
================================================================================

All documents available in relay-server-nodejs/:

Problem                          Solution
================================================================================
Can't connect to database        → Check .env credentials (SETUP.md)
Port 3000 already in use        → Change PORT in .env
npm install fails               → Clear cache: npm cache clean --force
Tests failing                   → Ensure database exists
Docker won't start              → docker-compose down -v (reset volumes)
API returns 401 Unauthorized    → Verify AUTH_TOKEN header
Frames not showing              → Check camera is streaming (POST /status)
Audio not working               → Verify audio polling every 150ms
